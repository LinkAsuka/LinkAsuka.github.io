基础文件配置

文件介绍

package.json

package.json 可以理解为这个项目的描述文件,可以重点关注的有

scripts 启动文件项目 dependencies 项目运行依赖,参与打包 devDependencies 开发依赖,辅助开发,不参与打包
package-lock.json

可以理解为npm的锁,第一次安装生成后不手动修改版本号,就不会自动去升级版本号

.browserslistrc

配置浏览器兼容

.editorconfig

配置编辑器,比如是否去掉后面空格啊,代码两行缩进啊

.eslintrc.js

配置插件和代码格式规则配置

.gitignore

设置可以不提交到git里的文件

文件修改

新增


assets/

  1. assets/fonts 摆放字体
  2. assets/images 摆放图片
  3. assets/scss 摆放scss

assets/scss/

  1. reset.scss 初始化css文件
    初始化css文件
  2. icon.scss 设置图标文件 详情看代码,这个太多了没必要
  3. base.scss 基础的全局css样式
    base配置
  4. index.scss 引入文件,引入了reset icon base
    引入文件
  5. minxi.scss css的使用函数,比如背景的media判断是用什么图片
    minxi的使用
  6. variable.scss 设置css字体大小和颜色的函数
    variable配置

设置index.scss的初衷就是因为这些都是全局文件,几乎所有的页面都要使用,所以要设置在在mian.js文件中,为了能够统一管理,在全局加新的css的时候看起来没有那么蠢,这里不推荐三个文件都在App页面写调用呢,亲亲.在mian.js新增全局css

这时候就有问题了,难道字体颜色和图标就不是全局用的了吗,为啥你的index没有呢
因为这两个文件有一个共同点,就是他们都是css的模块话,并不是具体的css样式,如果要使用里面的变量或者函数只能局部的去引入
这样很麻烦,那咋办呢.可以通过配置vue.config.js来实现,把这两个文件配置上就相当于全局引入了这两个配置文件

vue.config.js

PS:如果发现引入了这个文件还咩有用,不要慌.去看看你的 package.json里的sass-loader版本是不是太低啦
