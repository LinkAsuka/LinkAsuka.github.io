歌手列表的组件一共有两个点需要注意

  1. 滑动的时候,首字母置顶.滑动到下一个区间的时候,显示下个首字母
  2. 右侧的首字母导航要根据区间的变化颜色变化,当点击首字母的时候滑动到相应位置,滑动屏幕时候也可以滑动到相应位置
我们先来写 1

  1. 滑动的时候,首字母置顶.滑动到下一个区间的时候,显示下个首字母
    新建一个钩子文件src/components/index-list/use-fixed.js
    我们在组件的vue文件里,导入这个文件
    import useFixed from './use-fixed'
    在setup()中,拿到use-fixed文件return过来的值
    const { groupRef, onScroll, fixedTitle, fixedStyle, currentIndex } = useFixed(props)
    这样接收,根据setup()的规定还要记得return一下

思路大概就是通过ref拿到ul的dom,然后设置为groupRef.取它的子元素groupRef.value.children
通过循环和{li的dom.clientHeight拿到区间数组
拿到滑动的地方的坐标,然后和区间数组进行对比,通过区间内的循环的i值拿到当前位置在数据中的索引
在后台传过来的数据中找到i的那个数据,拿到它的title返回给组件
我们监听滑动的同时可以拿到组件的顶部距离上部分的距离,当这个距离小于li的高度时,给这个玩意添加一个被向上顶的动画效果当前li底部 - 滚动的Y值就是偏移量

src/components/index-list/use-fixed.js
我们再来写2

  1. 右侧的首字母导航要根据区间的变化颜色变化,当点击首字母的时候滑动到相应位置,滑动屏幕时候也可以滑动到相应位置
    新建一个钩子文件src/components/index-list/use-shortcut.js
    我们在组件的vue文件里,导入这个文件
    import useShortcut from './use-shortcut'
    在setup()中,拿到use-fixed文件return过来的值
    const { shortcutList, scrollRef, onShortcutTouchStart, onShortcutTouchMove } = useShortcut(props, groupRef)

这里主要的思路是先遍历后台给的数据,拿到一个所有开头的数组,然后把这个数组返回给组件页面进行循环和写样式
从上一个功能里我们的setup是可以拿到当前说在位置的索引的,所以这里我们也拿到这个索引然后直接和循环的index做对比,如果他们相等的话就可以改变导航条的字母颜色:class="{'current':currentIndex===index}"做到这里这个导航就已经人模狗样的了,然后我们开始做具体的滑动到相应区域的功能
在列表的外层div放事件,并给事件起名字,记住要加.stop.prevent防止冒泡
touchstart 触摸开始，多点触控，后面的手指同样会触发 onShortcutTouchStart
touchmove 接触点改变，滑动时 onShortcutTouchMove
touchend 触摸结束，手指离开屏幕时
先写一个跳转到相应位置的事件scrollTo,通过开发库的方法scrollToElement进行跳转.这个方法主要要的是拿到点击对应的li的dom,也就是要拿到索引的index
当onShortcutTouchStart触发的时候,直接点击就直接可以通过神秘的data-index方法拿到当前的index传过来就哦了
当onShortcutTouchMove触发的时候,拿到偏移的坐标然后加上索引向上取整,也就能拿到index
拿到之后直接调用scrollTo方法,跳转到索引位置就完事了
