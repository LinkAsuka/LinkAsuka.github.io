歌手详情

需要写一个从歌手列表点进去的时候看到这个歌手热门歌曲,这种功能一般都是拿到歌手列表点击过来的id传过来用id调用详情接口拿到值,这里我们通过二级路由的方式去实现
先新建一个歌手详情的页面/src/views/singer-detail.vue
然后我们去route去整一个娃,

route

  1. 歌手信息数据获取
    在歌曲列表页面整个跳转,跳转到二级页面的详情页面去
    我们要从列表的组件被点中的数据传给singer.vue,然后点击的时候,再把数据传给详情页面
    在列表组件页面传出来,写在点击事件哦
    emit('select', item)
    在歌手页面用selectSinger拿到
    <index-list @select="selectSinger"></index-list>
    为了写路由跳转整一个selectedSinger配合 <router-view :singer="selectedSinger">
    然后写跳转路径
    selectSinger(singer) { this.selectedSinger = singer this.$router.push({ path:/singer/${singer.mid} }) }
    最后在详情页面接收,并且this.singer传给接口拿到值

  2. 歌手歌曲列表信息获取
    平平无奇的引入import { processSongs } from '@/service/song'
    普普通通的拿到数据 this.songs = await processSongs(result.songs)
    result是上面歌手信息的接口的信息,就简简单单的拿到了

这样的歌曲列表,上面在弄个图片的情况挺多的,所以就写个组件出来吧

然后我们新建一个组件src/components/music-list/music-list.vue
看看我们这个组件需要拿到哪些数据呢

  1. 歌手名字 title
  2. 歌单 songs
  3. 歌手图片 pic
    写个点击事件this.$router.back()处理一下返回歌手列表的按钮,设置一下歌手详情列表在往上滑动的时候图片收缩的和下拉的时候图片放大的动画效果在计算属性里计算亿下下,把效果直接return给style

页面写到这里就大致出来了,但是刷新页面发现没有id可以传过来,拿不到id的话数据也出不来了,所以我们就用缓存的方式拿到id,这样刷新的时候也可以保留id调用接口了.这里我们需要用到一个插件
good-storage插件
缓存插件,为啥要进行一个封装呢,因为这个可以缓存一些对象啥的,而不是只能缓存字符串

传给设置的GSsinger,这样就不怕在页面刷新啦
