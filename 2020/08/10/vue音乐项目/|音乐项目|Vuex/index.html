底层播放铺垫

一般的音乐项目无论是鹅厂还是猪厂还是蓝狗子,你在看他的APP的时候都能发现他们的音乐APP下面会有一个小小的播放条,点开播放条可以看到当前播放的音乐或者是待播放的列表
这个东西无论是哪个tab和页面都是不会消失的,这种所有页面几乎都要用到的东西我们称之为全局组件,一般有这种情况的话我们会同过vue自带的一个全局组件工具vuex来实现
换句话说只有组件之间需要共享的数据才需要存到vuex中,这些数据会根据变化而变化

vuex的实现原理

Vuex从文档来看的话,主要要知道的是State, Getter, Mutation, Action

demo:

vuex-demo

https://gitee.com/Jessica_1994/vuex-demo.git

  1. State 就是一个全局的数据仓库
    类似于vue中的data设置一个全局的数据,在组件里要使用state的数据需要设置store:store 就可以拿到

  2. Mutation 可以改变数据
    用this.$store.commit()调用
    使用this.$store.state.meg 就能够拿到state里的meg数据

  3. Getter 是相当于一个计算属性,可以返回修改后的数据
    用法也和计算属性一样
    调用的数据可以通过this.$store.getter.formart

  4. Action 相当于可以对Mutation做一些封装
    mapState state的映射射的是封装的方法mapMutations, , mapActions, mapGetters
    引入Action相关的方法

    1.mapState(…mapState([‘count’]))
    这样映射的是数据,一般在计算属性里写,然后我们直接 {{ count }} 就能映射出里面的值
    2.mapGetters(…mapGetters([‘showCount’]))
    这样映射的是计算属性,一般在计算属性里写,然后我们直接用 {{ showCount }} 就可以拿到store中的showCount计算属性return的数据了
    3.mapMutations(…mapMutations([‘add’, ‘addN’]))
    这样映射的是方法,一般在方法里写,然后我们直接:click 当然我们还想传一些值@click="addN(3)"这样写在那边多写一个接收就完事了
    4.mapActions(…mapActions([‘asyncAdd’, ‘asyncNAdd’]))
    这样映射的是封装的方法,一般在方法里写

    直接调用addN来加,实现了异步调用 在外面调用封装的方法this.$store.dispatch('asyncSub')
