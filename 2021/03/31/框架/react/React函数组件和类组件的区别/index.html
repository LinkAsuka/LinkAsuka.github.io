函数组件和类组件

一般我们写页面,是选函数组件好还是类组件好呢
类组件创建了就会放在那,函数组件关掉的话就销毁了
所以我们一般拿函数组件来写组件,类组件来写页面

举个栗子

我们设置一个这样的场景,我有一个关注列表,用select框代表,选择关注的人然后点击关注按钮,就可以关注他,弹一个窗出来告诉你你关注了谁
这个场景在React中怎么写
先搭建一个父组件承载select框,然后我们写两个子组件分别作为函数组件和类组件的关注按钮和关注后的弹窗功能

  • ProfilePageFunction 函数组件的子组件
  • ProfilePageClass 类组件的子组件

不同

先点击我们要用类组件和构造函数显示

  • 点击类组件,初始值是王大拿,我们选择张大喵之后弹窗关注了张大喵
  • 点函数组件,初始值是王大拿,我们选择张大喵之后弹窗关注了王大拿

看出什么不同了吗
在拿到弹窗数据的时候类函数是这样写的alert('关注 ' + this.props.user);构造函数的组件是这样的alert('关注 ' + props.user);
问题就出在这个this上了,this是可以在render中取到最新的值的,而用props.user属实算是全班出去吃完零食回来你还不知道他们偷偷去吃零食了
如果我们坚持要用构造函数的话,是可以在this.props里面去拿的,但是不能说是毫无用处,简直是一点用处没有
只能用闭包来解决问题

函数组件改

原因

由于 props 是 Example 函数作用域下的变量，可以说对于这个函数的每一次调用中，都产生了新的 props 变量，它在声明时被赋予了当前的属性，他们相互间互不影响。换一种说法，对于其中任一个 props ，其值在声明时便已经决定，不会随着时间产生变化。
