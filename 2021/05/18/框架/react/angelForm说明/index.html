  1. 设置打开模态框

    1. 全部变成已读功能this.dialogRef1.current.showModelRef('detail',record,'readOnly');
    2. 传回来表单里没有的值(record所有的值都传回来)this.dialogRef1.current.showModelRef('edit',record,'saveOther');
    3. 传回来表单里没有的值(指定record的id传回来)this.dialogRef1.current.showModelRef('edit',record,['id']);
    4. 如果写那种模态框点击之后又弹出一个表格选择表格放入第一个模态框数据但是不改变其他 input 啥修改的东西
      怎么办呢,新增了editModelRef这个功能.用法:
      this.dialogRef1.current.editModelRef({...obj},"saveOther");
      用这个事件调用的话就不会影响组件里其他的东西只改变当前传过去的数据,而且像这种功能一般都会有一些不会显示出来的值比如 id 之类的,所以一般都和saveOther配合使用
    5. 拿到表单返回的值

  2. 设置关闭模态框 (在接口模式下不用写这个)
    一般来说在接口调用成功的时候写这个玩意,因为默认会打开表单提交的 loading 效果,接口调用成功 loading 也结束,完美
    this.dialogRef1.current.hideModelRef();
